# 要件定義書：属性仕分け工場

**アプリID**: `zokusei`
**対象年齢**: 7〜9歳（小学1〜3年生）
**ジャンル**: 知育パズル／集合・ベン図
**作成日**: 2026-03-23
**バージョン**: 1.0.0

---

## 1. アプリ概要

### 1.1 コンセプト

ベン図（集合）の概念を、フリック操作で直感的に学ぶ仕分けゲーム。
「赤い」「丸い」「生き物」などの属性を持つアイテムが上から流れてきて、子どもが正しい円の領域へ仕分けることで「AND（両方）」「OR（どちらか）」「NOT（どちらでもない）」の集合論的思考の基礎を自然に身につける。

### 1.2 学習目標

| 段階 | 学習内容 |
|------|---------|
| 初級 | 単一属性の分類（赤いもの vs 青いもの） |
| 中級 | 2属性の重なり（赤い かつ 丸い = 交差領域） |
| 上級 | 複合属性＋否定（生き物 でない もの = 外側領域） |

### 3.3 認知負荷を下げる設計ルール

1. **属性ラベルは最大10文字以内**（例：「あかい」「まるい」）
2. **フォントサイズ最小 32px**（視認性確保）
3. **アイテムの同時表示は1個のみ**（注意分散を防ぐ）
4. **フリック判定距離は最小30px**（誤操作を防ぐ）
5. **ゾーン境界はハイライト表示**（フリック中にターゲットが光る）
6. **音声読み上げ**: アイテム出現時に属性を読み上げ（例：「あかくて、まるい りんご」）

## 4. スコアリングロジック

### 4.1 基本スコア計算

```
基本点 = 正解時のベース点 × ゾーン難易度係数 × 反応時間ボーナス係数
```

| 要素 | 計算式 |
|------|--------|
| ベース点 | 100点（全ステージ固定） |
| ゾーン難易度係数 | `zone-left/right`: ×1.0, `zone-both`: ×1.5, `zone-none`: ×1.2 |
| 反応時間ボーナス | 2秒以内: ×1.5 / 4秒以内: ×1.2 / それ以上: ×1.0 |

**例**: 交差ゾーンへ1.8秒でフリック成功
→ `100 × 1.5 × 1.5 = 225点`

### 4.2 コンボシステム

```js
// コンボ倍率テーブル
const COMBO_MULTIPLIER = {
  1: 1.0,   // 1連続: 通常
  2: 1.2,   // 2連続: ×1.2
  3: 1.5,   // 3連続: ×1.5
  4: 2.0,   // 4連続: ×2.0
  5: 3.0,   // 5連続以上: ×3.0（最大）
};

// コンボ加算スコアの例
totalScore += Math.floor(baseScore * comboMultiplier);
```

**コンボ継続条件**:
- 正解ごとに `combo++`
- 不正解でリセット（`combo = 0`）
- タイムアウト（アイテムが通過）でもリセット

**コンボ演出**:
| コンボ数 | 表示 | 効果音 |
|---------|------|--------|
| 2連続 | 「ナイス！×2」 | ポーン（高め） |
| 3連続 | 「すごい！×3」🔥 | ファンファーレ短め |
| 5連続以上 | 「さいこう！×5」🌟炎 | ファンファーレ長め＋画面フラッシュ |

### 4.3 ステージクリア評価

```
正答率 = 正解数 ÷ 出題数 × 100

★★★: 正答率 80% 以上
★★☆: 正答率 60% 以上
★☆☆: 正答率 40% 以上
（クリア失敗）: 正答率 39% 以下 → リトライ推奨ダイアログ
```

### 4.4 ハイスコア管理

```js
// localStorage キー設計
localStorage.setItem(`zokusei_stage_${stageId}_best`, JSON.stringify({
  score: 1850,
  stars: 3,
  maxCombo: 7,
  clearedAt: "2026-03-23"
}));
```

## 5. 難易度設計（ステージ進行）

### 5.1 難易度パラメータ一覧

| パラメータ | 初級(Lv1-3) | 中級(Lv4-6) | 上級(Lv7-10) |
|-----------|------------|------------|-------------|
| 属性数/円 | 1つ | 1〜2つ | 2〜3つ |
| アイテム数/ステージ | 8個 | 12個 | 16個 |
| 降下速度（秒/画面） | 4.0s | 2.8s | 1.8s |
| 交差ゾーン使用 | なし | あり | あり |
| 「どちらでもない」使用 | なし | 20%混入 | 40%混入 |
| 属性の組み合わせ | 単純対比 | 2軸交差 | 多軸＋否定 |

### 5.2 ステージ別属性テーブル

```
[Lv1] 「あかい」vs「あおい」
  ─ 左円: あかいもの | 右円: あおいもの
  ─ 交差なし, 外側なし
  ─ アイテム例: 赤いリンゴ / 青い風船 / 赤いいちご

[Lv2] 「まるい」vs「しかくい」
  ─ 左円: まるいもの | 右円: しかくいもの
  ─ 交差なし（形状は排他的）
  ─ アイテム例: ボール / 積み木 / 地球儀

[Lv3] 「いきもの」vs「たべもの」
  ─ 左円: いきもの | 右円: たべもの
  ─ 外側: どちらでもないもの（乗り物など）が初登場
  ─ アイテム例: ねこ / りんご / 電車（外側）

[Lv4] 「あかい」×「まるい」（2属性交差・初）
  ─ 左円: あかいもの | 右円: まるいもの
  ─ 交差: あかくて まるいもの（赤いりんご, 赤いボール）
  ─ 属性の組み合わせを視覚的に説明するチュートリアルを挿入

[Lv5] 「うごく」×「みずのなか」
  ─ 左円: うごくもの | 右円: みずのなかにいる
  ─ 交差: 水中で泳ぐ（魚, イルカ）
  ─ 左のみ: 陸上動物 / 右のみ: 水草, サンゴ

[Lv6-8] 複合属性（2属性×2）
  ─ アイテムが複数の属性を同時に持つ
  ─ 例: 「赤い かつ 四角い」→ 交差ゾーンへ
  ─ 降下速度UP + アイテム数増加

[Lv9-10] 上級：3属性＋否定
  ─ 円のラベルが「あかくない」「〜ではないもの」など否定形登場
  ─ 降下アイテムが最大3属性を保有
  ─ 制限時間モード（ステージ全体に90秒タイマー追加）
```

### 5.3 アダプティブ難易度調整（任意実装）

```js

  currentSpeed *= 1.1;  // 降下時間を伸ばす = 遅くなる
```

## 6. エラーハンドリング（誤答フィードバック）

### 6.1 フィードバック体系

```
誤答発生
  │
  ├─ [視覚] アイテムが赤く点滅（200ms × 3回）
  │         正解ゾーンへ矢印アニメーション（0.5秒）
  │         誤答ゾーンを薄赤でハイライト
  │
  ├─ [音声] 「あれ？ちがうよ」（低めのトーン、短い効果音）
  │         正解ゾーンの属性を音声で再読み上げ
  │
  └─ [触覚] デバイスのバイブレーション（短く1回, 50ms）
```

### 6.2 フィードバックの詳細仕様

| 事象 | 視覚 | 音声 | テキスト |
|------|------|------|---------|
| **正解** | アイテムが消えてキラキラパーティクル | 明るいポーン音 | 「ピッタリ！」 |
| **不正解** | アイテム赤点滅 → 正解ゾーンへ飛ぶアニメ | 低いブー音 | 「ちがうよ、こっちだよ！」 |
| **タイムアウト** | アイテムが下へ落下・消滅 | 落下効果音 | 「おちちゃった…」 |
| **コンボ途絶** | コンボ表示が割れるアニメ | ガラス割れ音 | —（テキスト表示なし） |
| **連続3回誤答** | 工場長ロボットがヒントを表示 | 「おしえてあげるね！」 | 属性の説明吹き出し |

### 6.3 ヒントシステム

```
連続誤答 3回 → ヒントモード発動
  ・正解ゾーンに光るリング（ゆっくり点滅）を常時表示
  ・次のアイテムに属性ラベルを直接オーバーレイ表示
  ・ヒント発動中はスコアに0.5倍ペナルティ

連続誤答 5回 → 簡易チュートリアル再表示
  ・「いっかいれんしゅうしよう！」ダイアログ
  ・そのアイテムを使って正解をデモ（自動アニメ）
  ・「わかった！」タップで再開
```

### 6.4 誤答後のアイテム処理

```js
// 誤答時の処理フロー
function onWrongAnswer(item, selectedZone, correctZone) {
  // 1. 赤点滅アニメーション
  item.classList.add('shake-error');

  // 2. 正解ゾーンへ飛ぶアニメーション（800ms）
  animateTo(item, correctZone, duration=800);

  // 3. 正解ゾーンが光る
  highlightZone(correctZone, color='#FFD700', duration=600);

  // 4. コンボリセット
  state.combo = 0;

  // 5. スコア加算なし（減点はしない：子どもへのストレス回避）

  // 6. ログ記録
  logAction(item, selectedZone, correctZone, isCorrect=false);
}
```

> **設計方針**: 減点（マイナス）は行わない。不正解はスコア加算なし＋コンボリセットのみ。過度なペナルティは7〜9歳の学習意欲を下げるため。

### 8.3 フリック判定ロジック

```js
// ゾーン判定: フリック終点の座標からゾーンを決定
function getZoneFromPoint(x, y) {
  const inLeft  = isInsideCircle(x, y, LEFT_CIRCLE);
  const inRight = isInsideCircle(x, y, RIGHT_CIRCLE);

  if (inLeft && inRight)   return 'zone-both';
  if (inLeft)              return 'zone-left';
  if (inRight)             return 'zone-right';
  return 'zone-none';
}

// 最低移動距離: 30px 以上でフリックと認識
const MIN_FLICK_DISTANCE = 30;
```

### 8.4 アクセシビリティ対応

- **文字サイズ**: 全テキスト最小 `2rem`（~32px）
- **タッチターゲット**: 最小 `48×48px`（Apple HIG 準拠）
- **色のみへの依存禁止**: 正誤は色＋形＋アニメーションの3要素で表現
- **音声ON/OFFトグル**: 設定画面から切り替え可能
